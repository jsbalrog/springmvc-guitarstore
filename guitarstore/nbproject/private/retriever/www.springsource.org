      <h2 class="title">
      <a href="/node/2281">Overview of the Spring 3.0 Web Stack</a>
    </h2>
  
  
  
  
  
  <div class="content">
    <p> 
<div style="float: right; width:248px; padding: 10px">
<a href="http://www.infoq.com/presentations/Introducing-Spring-Roo"><img height="173" alt="Overview of Spring 3.0 Web Stack" width="248" border="0" src="/files/SpringOne2GX2009_SpringWebOverview.png" /></a>
</div>
Another <a href="http://www.infoq.com">Infoq</a> video presentation has been posted that gives an <a href="http://www.infoq.com/presentations/donald-overview-spring-3.0-web-stack">Overview of the Spring 3.0 Web Stack</a>. The session was recorded at last October's <a href="http://www.springone2gx.com">SpringOne 2GX</a> 2009 conference and features, <a href="http://blog.springsource.com/author/kdonald/">Keith Donald</a>, <a href="http://www.springsource.com">SpringSource</a>'s web guru. Keith provides some quick notes for viewers:
</p>
<p>
<ul>
<li>The December GA release of Spring 3.0 shipped the additional Spring MVC simplifications predicted in this talk, including:
  <ul>
    <li>A dedicated 'mvc' namespace for simplifying Spring MVC setup</li>
    <li>Automatic registration of type converters based on what is present in your classpath. This includes automatic setup of JSON (Jackson), XML (JAXB), and Joda Time support.</li>
    <li>Support for annotation-driven formatting with @NumberFormat and @DateTimeFormat</li>
  </ul>
</li>
<li>Community members interested in these capabilities can read more in this <a href="http://blog.springsource.com/2009/12/21/mvc-simplifications-in-spring-3-0">blog entry</a>. The <em>petcare</em> sample application referenced in this presentation can also be accessed in the <a href="https://src.springframework.org/svn/spring-samples/petcare/trunk">Spring Samples SVN repository</a>. This sample app is directly importable & runnable with <a href="http://www.springsource.com/products/sts">SpringSource Tool Suite</a>.
</li>
</ul>
</p>

      <h2 class="title">
      <a href="/groovy-eclipse-2-0-0-released">Groovy-Eclipse 2.0.0 Released</a>
    </h2>
  
  
  
      <div class="submitted">
      Submitted by Christian Dupuis on Fri, 2010-01-15 13:51    </div>
  
      <div class="taxonomy"> in <ul class="links inline"><li class="taxonomy_term_4 first last"><a href="/taxonomy/term/4" rel="tag" title="News and announcements
">News and Announcements</a></li>
</ul></div>
  
  <div class="content">
    At the start of May 2009 we announced we were working on a new approach to joint compilation for mixed Java/Groovy projects in Eclipse. We are pleased to now announce the final release of Groovy-Eclipse v2.0.0, based on that new technology. During the months of development we have rebased (and almost entirely rewritten) version 1 of the Groovy-Eclipse plugin, with the goal of offering first class Groovy support in the Eclipse IDE, comparable to the experience Java developers have in Eclipse.

The most important new features that Groovy-Eclipse 2.0.0 provides are:

<ul>
<li>a new approach to compiling joint Groovy-Java code that does not require a
stub compiler</li>
<li>advanced and extensible content assist</li>
<li>vast speed and memory improvements over previous releases</li>
<li>debug support</li>
<li>deep integration with the Java Development tools</li>
</ul>

Also, over 380 bugs have been addressed since the previous release.

See the <a href="http://docs.codehaus.org/display/GROOVY/Groovy-Eclipse+2.0.0+New+and+Noteworthy" >release notes</a>. 

The Eclipse update site is available here:

<ul>
<li>http://dist.springsource.org/release/GRECLIPSE/e3.5/</li>
</ul>

Please comment on the <a href="http://docs.codehaus.org/display/GROOVY/Groovy-Eclipse+Roadmap" >roadmap</a> if you want a say in the planning of future releases.  </div>

      <h2 class="title">
      <a href="/node/2277">dm Server project moves to Eclipse.org</a>
    </h2>
  
  
  
      <div class="submitted">
      Submitted by Adam Fitzgerald on Tue, 2010-01-12 10:30    </div>
  
      <div class="taxonomy"> in <ul class="links inline"><li class="taxonomy_term_4 first last"><a href="/taxonomy/term/4" rel="tag" title="News and announcements
">News and Announcements</a></li>
</ul></div>
  
  <div class="content">
    <p><a href="http://www.springsource.com">SpringSource</a> and the <a href="http://eclipse.org">Eclipse Foundation</a> have just <a href="http://blog.springsource.com/2010/01/12/dm-server-project-moves-to-eclipse-org/">announced</a> a new project proposal, <a href="http://eclipse.org/proposals/virgo/">Project Virgo</a>, whose primary goal is to provide a runtime platform for the development of server-side enterprise applications built on top of Equinox. The project will become the new home for <a href="http://www.springsource.com/products/dmserver">SpringSource dm Server</a> and will be a part of the <a href="http://www.eclipse.org/rt">Eclipse Runtime (RT)</a> project. Other dm Server related projects, such as <a href="/bundlor">Bundlor</a> and the <a href="/dmservertools">dm Server Eclipse tools</a> will also move to over to the Eclipse community as part of the proposal. <a href="http://blog.springsource.com/author/adrianc/">Adrian Colyer</a>'s <a href="http://blog.springsource.com/2010/01/12/dm-server-project-moves-to-eclipse-org/">blog post</a> covers all the details about the move and the Eclipse Foundation's executive director, <a href="http://dev.eclipse.org/blogs/mike/">Mike Milinkovich</a> also weighs in with his <a href="http://dev.eclipse.org/blogs/mike/2010/01/12/one-big-step-for-eclipsert/">perspective</a>.</p>
<p>The project proposal coincides with the community final release today of the 2.0.0 version of dm Server. </p>

      <h2 class="title">
      <a href="/node/2273">Spring Roo 1.0.0 (GA) Released</a>
    </h2>
  
  
  
      <div class="submitted">
      Submitted by Ben Alex on Thu, 2009-12-31 02:36    </div>
  
  
  <div class="content">
    <p>We're delighted to announce the general availability (GA) of <a href="http://www.springsource.org/roo">Spring Roo</a> 1.0.0.</p>
<p>Spring Roo is a next-generation rapid application development tool for Java developers. With Roo you can easily build full Java applications in minutes. It differs from other tools by focusing on:</p>
<ul>
<li><em>Higher Java productivity</em>: Try the ten minute test and see for yourself.</li>
<li><em>Stock-standard Java</em>: Roo uses the Java APIs and standards you already know and trust.</li>
<li><em>Usable and learnable</em>: Roo features an extremely high level of usability and an advanced shell.</li>
<li><em>No engineering trade-offs</em>: Roo has no runtime portion and does not impose any CPU, RAM or disk storage cost.</li>
<li><em>Easy Roo removal</em>: Roo can be easily removed from a user project in under five minutes.</li>
</ul>
<p>This release follows some eight community releases (A1, A2, M1, M2, RC1, RC2, RC3, RC4) and extensive feedback. Before reaching Roo 1.0.0 we already had <a href="http://forum.springsource.org/forumdisplay.php?f=67">over 1,000 forum posts</a> and some <a href="http://jira.springsource.org/browse/ROO">500 Jira issues</a> logged. There's already been <a href="http://www.jaxitalia.com/konferenzen/jaxitalia09/speaker/#4566">dozens</a> <a href="http://www.slideshare.net/krimple/philly-spring-ug-roo-overview">of</a> <a href="http://europe.springone.com/europe-2009/presentation/Extreme+Productivity+in+Application+Development">conference</a> <a href="http://www.springone2gx.com/conference/new_orleans/2009/10/session?id=15641">and</a> <a href="http://www.springone2gx.com/conference/new_orleans/2009/10/session?id=15644">JUG</a> <a href="http://thestrangeloop.com/sessions/extreme-web-productivity-spring-roo">presentations</a> <a href="http://www.therichwebexperience.com/conference/orlando/2009/12/session?id=16629">on</a> <a href="http://qconlondon.com/london-2010/speaker/Rod+Johnson">Roo</a>, several third-party add-ons and there's even people already writing a commercial book on Roo.</p>
<p>Spring Roo is thoroughly documented, with around 100 pages of detailed <a href="http://static.springsource.org/spring-roo/reference/index.html">documentation available online</a> and in the download ZIP. You can read a <a href="http://blog.springsource.com/2009/05/01/roo-part-1/">three-part</a> <a href="http://blog.springsource.com/2009/05/27/roo-part-2/">blog</a> <a href="http://blog.springsource.com/2009/06/18/roo-part-3/">series</a> introducing Roo, plus view recently-updated conference presentations (<a href="http://www.slideshare.net/benalexau/introduction-to-spring-roo-100-2805183">Introduction to Roo</a> and <a href="http://www.slideshare.net/benalexau/spring-roo-100-technical-deep-dive">Roo Technical Deep Dive</a>).</p>
<p>You can <a href="http://www.springsource.com/download/community?project=Spring%20Roo">download</a> this new version of Roo, plus access key links at:</p>
<ul><a href="http://www.springsource.org/roo">http://www.springsource.org/roo</a></ul>
<p>We hope you that enjoy using Roo. The Roo team will be taking a short break to celebrate the New Year, and we look forward to hearing about your Roo experiences and following up with more Roo resources. Happy New Year!</p>
  </div>
